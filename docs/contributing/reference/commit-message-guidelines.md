# Commit Message Guidelines

This repository uses [Conventional Commits](https://www.conventionalcommits.org/).

## Format

- Use `type:` or `type(scope):` as the prefix.
- Write the subject in imperative mood (for example: "fix nil check in webhook").
- Keep the subject concise and specific.

## Common Types

- `feat`: new behavior
- `fix`: bug fix
- `refactor`: behavior-preserving code restructure
- `test`: tests only
- `docs`: documentation only
- `chore`: maintenance or non-product changes
- `ci`: CI pipeline changes
- `build`: build/tooling/dependency changes
- `perf`: performance improvements
- `revert`: revert a previous commit
